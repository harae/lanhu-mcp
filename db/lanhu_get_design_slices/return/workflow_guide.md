# 切图下载工作流

## 总要求
- 始终使用中文回复用户
- AI 必须主动完成整个流程，而不是只返回素材信息
- 参数来源于 `lanhu_get_design_slices` 的返回结果

## 执行步骤
1. 创建 TODO 计划，明确分析项目、确认目录、下载、校验、清理等阶段
2. 识别项目类型和资源目录约定
3. 根据图层路径、父层级、尺寸生成语义化文件名
4. 检测当前操作系统和可用下载工具
5. 选择最合适的下载方案执行批量下载
6. 校验下载结果，清理临时文件，更新 TODO

## 目录选择规则
- 优先使用用户明确指定的输出目录
- 其次使用项目现有约定目录，例如 `src/assets/`、`assets/images/`
- 如果项目没有明显约定，则使用 `design_slices/{design_name}/`

## 命名规则
- icon：`icon_xxx.png`
- background：`bg_xxx.png`
- button：`btn_xxx.png`
- 其他资源基于 `layer_path`、`parent_name` 和尺寸生成语义名

## 环境检测原则
- 不要假设任何工具一定可用，必须先检测
- 优先系统内置工具，例如 `curl`、PowerShell、`wget`
- 如果系统工具不可用，再考虑 `python`、`node` 等环境
- 只有在必要时才生成临时脚本

## 执行检查项
- 检测操作系统类型
- 检测可用下载工具
- 控制并发下载
- 清理临时文件
- 不要在最终代码里保留蓝湖远程 CDN 地址
